# Issues
- Criarmos uma nova funcionalidade ou um novo problema.

# Pull Requests
- Solicitar a entrega de uma mudança
  - Enviar novos códigos e alterações pra um repositório PÚBLICO que não é meu ou PRIVADO que eu faço parte.
- Existe revisão das mudanças antes delas entrarem no código principal.
- Cada pull requests tem registro das mudanças, documentação dos commits, texto explicativo e afins. (BEM SIMILAR AS ISSUES.)

## Exemplo prático

1. Nós trabalhamos na Rocketseat
2. Você cria uma branch pra desenvolver login (feature/login)
3. Você faz suas alterações, implementações e afins
4. Abre um PR pra **Main**
5. Eu consigo revisar o PR e aprovar ele
6. Ocorre um merge da feature/login -> main

## Benefícios
- Descrição clara
- Pequenos PRs
- Checklist de testes ou etapas a serem realizadas

- Google Drive -> Link fica público (qualquer pessoa da internet pode acessar) -> Qualquer pessoa pode editar.
- Github -> Repositório que fica público -> Eu não consigo enviar e alterar o código do repositório se ele não for meu -> Abrir uma PULL REQUEST
